# 📦 SourceDump

**SourceDump** — это легковесная Python-утилита для подготовки структуры проекта и, при необходимости, содержимого файлов в удобный текстовый формат для LLM.

По умолчанию утилита **строит только дерево проекта**.  
Чтобы добавить содержимое файлов, используйте флаг **`-f` / `--full`**.

---

## ✨ Основные возможности

- **Tree-first режим по умолчанию**  
  Без дополнительных флагов SourceDump выводит только структуру проекта — это удобно для быстрого обзора и экономии токенов.

- **Полный дамп по флагу `--full`**  
  При включении `-f` / `--full` в вывод добавляется содержимое файлов в XML-подобных блоках:

```xml
  <file path="project/file.py">
  ...
  </file>
```

* **Фильтрация мусора и системных директорий**
  По умолчанию исключаются:
  `.git`, `__pycache__`, `node_modules`, `.DS_Store`, `uv.lock`, `.venv`

* **Пропуск бинарных файлов**
  Скрипт старается не включать бинарные файлы в дамп, чтобы не засорять контекст.

* **Режим Diff Prompt (`-dp`/`--diff-prompt`)**
  Добавляет системную инструкцию для LLM, чтобы модель отвечала **только в формате Unified Diff**.

* **Оптимизация токенов (`-s`/`--strip`)**
  В режиме `--full` можно удалить пустые строки из содержимого файлов, чтобы уменьшить размер контекста.

* **Подсчёт токенов**
  Если установлен `tiktoken`, токены считаются точнее.
  Иначе используется приблизительная оценка.

---

## 🚀 Быстрый старт

### Только дерево проекта

```bash
uv run SourceDump.py
```

### Дерево + содержимое файлов

```bash
uv run SourceDump.py -f
```

### Полный дамп для LLM с экономией токенов

```bash
uv run SourceDump.py -f -s -o prompt.md
```

### Полный дамп + инструкция отвечать только диффами

```bash
uv run SourceDump.py -f -s -dp -o prompt.md
```

### Использование файла исключений

```bash
uv run SourceDump.py -f -ef .gitignore -o context.md
```

### Несколько путей + дополнительные исключения

```bash
uv run SourceDump.py ./src ./tests -f -e "*.log" "build" "dist" -o project.txt
```

## 📖 Параметры запуска

| Флаг      | Полное имя       | Описание                                                                          |
| :-------- | :--------------- | :-------------------------------------------------------------------------------- |
| `paths`   | —                | Пути для анализа. Если не указаны, используется текущая директория `.`            |
| `-f`      | `--full`         | Добавить содержимое файлов. Без этого флага выводится только дерево проекта       |
| `-o`      | `--output`       | Сохранить результат в файл. Если не указан — вывод в консоль                      |
| `-e`      | `--exclude`      | Дополнительные шаблоны исключений                                                 |
| `-ef`     | `--exclude-file` | Файл со списком шаблонов исключений                                               |
| `-dp`     | `--diff-prompt`  | Добавить в начало дампа инструкцию для LLM отвечать только Unified Diff           |
| `-s`      | `--strip`        | Удалить пустые строки из содержимого файлов. Имеет смысл только вместе с `--full` |

---

## 📝 Формат файла исключений

Файл, переданный в `--exclude-file`, читается **построчно**.

Поддерживаются:

* обычные имена директорий и файлов
* glob-шаблоны вроде `*.log`
* строки-комментарии, начинающиеся с `#`

Пример:

```text
# directories
.git
node_modules
build
dist

# files
*.log
*.tmp
.env
```

> Важно: это **упрощённые шаблоны**, а не полная реализация синтаксиса `.gitignore`.

## 📤 Формат вывода

### По умолчанию

Скрипт выводит только дерево проекта:

```text
=== PROJECT STRUCTURE ===

Root: my_project/
├── src
│   └── main.py
└── README.md
```

### В режиме `--full`

После дерева добавляются блоки с содержимым файлов:

```text
=== FILE CONTENTS ===

<file path="my_project/src/main.py">
print("hello")
</file>
```

---

## 📊 Подсчёт токенов

После генерации дампа скрипт выводит примерное или точное количество токенов:

* **точно** — если установлен `tiktoken`
* **примерно** — если `tiktoken` недоступен

Это помогает заранее понять, поместится ли дамп в контекст LLM.

---

## ⚠️ Важные замечания

* `--strip` влияет только на содержимое файлов и бесполезен без `--full`
* бинарные файлы стараются автоматически пропускаться
* файл исключений не является полноценным `.gitignore`-парсером
* очень большие репозитории лучше дампить по частям: например, только `src`, `tests`, `docs`

## ✅ Типовые сценарии

### 1. Быстро показать модели структуру проекта

```bash
uv run SourceDump.py
```

### 2. Передать весь релевантный код в LLM

```bash
uv run SourceDump.py -f -o context.md
```

### 3. Подготовить промпт для безопасных правок через дифф

```bash
uv run SourceDump.py -f -s -dp -o prompt.md
```

### 4. Исключить временные артефакты и логи

```bash
uv run SourceDump.py -f -e "*.log" "*.tmp" "dist" "build"
```
